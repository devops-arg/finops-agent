# CLAUDE.md вҖ” FinOps Intelligence Platform

## What This Is

Demo project by **DevOps ARG** (www.devopsarg.com) вҖ” an AI-powered FinOps platform that analyzes AWS cloud costs and infrastructure using conversational AI. This is a showcase of DevOps ARG's capabilities in platform engineering, agentic AI, and cloud cost optimization.

**This is NOT a toy.** It demonstrates production-grade patterns: multi-round agentic reasoning, tool orchestration, SSE streaming, LocalStack for zero-cost demos, and a real FastAPI backend with proper session management.

## How to Run

### Demo mode (no AWS account needed)

```bash
cp .env.example .env
# Edit .env: set ANTHROPIC_API_KEY (or OPENAI_API_KEY)
# USE_LOCALSTACK=true is already the default
docker compose up --build
# Open http://localhost:3000
```

### Real AWS mode

```bash
# In .env: set USE_LOCALSTACK=false, add AWS credentials
# Minimum IAM: ce:Get*, sts:GetCallerIdentity (read-only)
docker compose up --build
```

## Reasoning Engine

The core differentiator. `backend/reasoning/engine.py`:

1. User query arrives
2. LLM gets SYSTEM_PROMPT + conversation history + 15 tool definitions
3. **Round 1**: LLM calls tools (e.g., `get_current_date` вҶ’ `query_aws_costs`)
4. **Reflection**: Engine injects reflection prompt вҖ” "do you have enough data?"
5. **Rounds 2-4**: Additional tool calls if needed (comparison periods, different breakdowns)
6. **Final synthesis**: Structured markdown answer with real numbers
7. If LLM "plans" instead of acting in round 1, engine pushes back: "execute, don't describe"

SSE events: `thinking`, `tool_call`, `tool_result`, `answer`, `done`, `error`

## Demo Scenario

The LocalStack seed (`scripts/seed_localstack.py`) simulates a **Series A LatAm fintech startup**:

- **EC2**: 3x m5.xlarge (prod API), 2x c5.2xlarge (prod workers), 2x m5.large (prod web), staging fleet, stopped dev instances
- **RDS**: db.r5.2xlarge primary + r5.xlarge replica (prod), t3.medium (staging)
- **ElastiCache**: r6g.large Redis (prod), t3.micro (staging)
- **S3**: 4 buckets (prod-data, backups, staging, ML models)
- **Secrets Manager**: DB passwords, API keys (demo values)

`report_data.json` contains pre-generated cost data matching this infra.

## Config

All via `.env` вҖ” see `.env.example` for full list. Key vars:

- `AI_PROVIDER`: `anthropic` or `openai`
- `ANTHROPIC_API_KEY` / `OPENAI_API_KEY`: LLM credentials
- `USE_LOCALSTACK`: `true` for demo, `false` for real AWS
- `PORT`: Backend port (default 8000)

## Code Conventions

- Python 3.11, FastAPI + Pydantic
- Abstract LLM provider interface вҖ” swap Claude/GPT without touching business logic
- Tool system: implement `BaseTool`, register with `ToolRegistry`, engine discovers automatically
- No ORM вҖ” direct boto3 calls to Cost Explorer / describe-* APIs
- Frontend: single `index.html`, vanilla JS, no framework, SSE via EventSource
