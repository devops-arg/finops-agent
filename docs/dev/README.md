# docs/dev/

Local-only working files generated by maintainers when scanning real AWS accounts.
Everything in this folder is **gitignored** — files here will never be pushed.

## Files you may see here

- `cleanup-plan.md` — human-reviewed waste plan from a real AWS scan, with
  account IDs and resource names. Generated by hand from `/api/findings`.
- `cleanup.sh` — the AWS CLI commands to remediate the plan above. Every command
  is commented out; the operator uncomments and runs section by section.

## Why these aren't in the public repo

They contain real customer data (account IDs, resource ARNs, EIPs, snapshot IDs).
Putting them in a public repo would leak inventory of a paying customer's
environment.

## Re-generating

There is no automation that writes these files today — they are produced by hand
during a real-account scan and reviewed before any deletion is performed. If
that changes, document the generator script here and add it to `scripts/`.
